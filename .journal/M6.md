## ADRs introduced

- **ADR-0010** (`docs/adr/0010-current-term-commit-rule.md`) — current-term
  commit rule (Raft Figure 8 fix). A leader advances `commitIndex` to `N`
  only when a majority report `matchIndex >= N` AND `log[N].term ==
  currentTerm`; prior-term entries commit indirectly; the quorum index is
  a `slices.Sort`'d snapshot of `matchIndex` values, never map-iteration
  (C-8). Retires P0-1.
- **ADR-0011** (`docs/adr/0011-log-storage-mirror.md`) — mirror the
  replicated log into the local Storage subset. `proposeLocked` and the
  AppendEntries receiver mirror every `Append` / `TruncateSuffix` into
  `n.storage` BEFORE any dependent response (REPL-09 / P0-4 final); the
  `pkg/raft.Storage` ↔ `pkg/storage.Storage` reconciliation is explicitly
  deferred to a Phase 7 ADR.

## Decisions made (non-ADR)

- **RATIFIED decision 1 — 50/150/300 ms default alignment.**
  `applyDefaults` was corrected toward the already-documented LLD §2
  contract (50 / 150 / 300), so this is a CONFORMANCE FIX, not a contract
  change — no RFC is owed (PROC-03 not triggered; RESEARCH Pitfall 1).
- **RATIFIED decision 3 — `proposeLocked` is package-internal,**
  surfaced through `raft.TestNode.Propose`; no public `Propose` until
  Phase 7. `TestNode` widened with `Propose` + four inspectors, all Phase-7
  cleanup targets.
- **`TestFigure8` scripts by SHAPE, not fixed node identities** —
  RNG-driven timeouts (ADR-0009) make a specific stage-winner
  non-reproducible; the safety property is identity-independent.
  External `package raft_test` defeats the raftest import cycle.
- **`ProposeToLeader` is a SEPARATE real-leader path** on
  `raftest.Cluster`; the Phase-4 recorder-shim `Propose` is byte-for-byte
  untouched, so `TestCluster_TwoRunsByteIdentical` stays byte-identical.
  Recorder path retired in Phase 7.
- **Quorum substitutes self=`LastIndex` in the snapshot loop only**
  (Pitfall 6) — it does NOT write `matchIndex[self]` (diverges from the
  redundant RESEARCH Pattern-3 map write).
- **`commitIndex` persistence stays best-effort / unasserted** (Pitfall 7);
  only log-entry durability is the hard P0-4 obligation.

## What surprised us

1. **An isolated leader keeps reporting `Role=Leader` at its stale term.**
   Network isolation does not demote a leader — `TestFigure8`'s first
   draft routed proposals via a cluster-wide first-leader scan, which sent
   them into the "crashed" (isolated) node. The fix was
   `driveUntilLeaderIn(minTerm, island...)`: scan only the connected
   island above a term floor. The lesson — partition is a delivery
   property, not a role property — is obvious in hindsight and cost an
   afternoon to find.
2. **The byte-identical recorder test was a tripwire worth keeping.** The
   instinct was to reuse the Phase-4 `Propose` recorder shim for the new
   real-leader proposals. That would have changed the recorded byte stream
   and broken `TestCluster_TwoRunsByteIdentical`. Splitting
   `ProposeToLeader` off as a separate path kept the byte-identity proof
   intact (verified `-count=2`) and surfaced that the recorder path is now
   dead weight to delete in Phase 7.
3. **P0-4 was "done" in Phase 5 but vacuous on the log side.** The vote
   half (HardState-before-vote) shipped in Phase 5; the entry half had no
   code path because the log lived only in memory. The raftdebug invariant
   would have passed trivially (nothing to check). Writing the *negative*
   ordering test first (Pitfall 8) forced the realisation that the mirror
   had to exist before the assertion meant anything.
4. **The current-term commit rule is one `&&`, and it is the whole phase.**
   The Figure-8 fix reduces to adding `&& log[N].term == currentTerm` to
   the quorum check. Everything else — the receiver, the response handler,
   the Storage mirror, the chaos harness — exists to *prove* that one
   conjunct is correct under churn. The asymmetry between how small the
   safety rule is and how much scaffolding it takes to trust it is the
   recurring Raft lesson.
5. **The local `golangci-lint` binary out-ran the pinned config.** The
   installed v2.12.2 rejects the repo's schema-v1 `.golangci.yml` (pinned
   for the CI binary per ADR-0003). `go vet` (the pre-commit gate) stays
   the local source of truth; the lint divergence is environmental, not a
   code finding — but it is a standing papercut Phase 14 (ADR-0003 update)
   should close.
6. **A "deterministic at a fixed seed" scripted test must key its setup off
   LIVE state, not a value captured earlier (06-07 addendum).** Post-close
   verification found `TestFigure8` flaky (~10% no-race, ~17-20% -race) — but
   *only* at SETUP preconditions, never a safety assertion (no-lost-commit
   stayed green via the 1000-seed chaos sweep). Two layers: a spurious
   `termA >= 2` floor rejecting a valid term-1 first leader, and `termA`
   captured at the first election no longer matching the term index-2 landed
   at after an RNG re-election of leaderA. Fix (test-only, no SUT change):
   track by SHAPE + relative terms discovered from the live elected leader —
   re-bind `termA` to leaderA's live term at the index-2 append instant. SC3
   re-closed: `TestFigure8 -count=50` green with and without `-race`. See
   `06-07-SUMMARY.md`.
