## ADRs introduced

- **ADR-0006** (`docs/adr/0006-fakeclock-determinism-model.md`) —
  Deterministic FakeClock Model. Locks: synchronous `Advance`
  (no background goroutine, no `runtime.Gosched`); FIFO same-instant
  tiebreak via monotonic `(expiry, seq)` heap key; 1s panic-on-stuck
  quiescence guard around the per-fire channel send (silent hang is
  worst-case test outcome); `Fake.mu` released around the send so
  consumer tick handlers can call back into Fake without deadlock.
  References LLD §3.
- **ADR-0007**
  (`docs/adr/0007-inproc-hub-chaos-seed-splitting.md`) — inproc Hub
  Chaos: Seed-Splitting + Single Dispatcher. Locks: single `int64`
  `Seed` split via `SHA-256(seed_be8 || tag)[:8]` into per-knob PCG
  sub-Sources (drop/delay/reorder/duplicate); tag strings are ABI;
  single dispatcher goroutine drains a `(deliverAt, seq)` heap under
  `h.mu`; partition install is symmetric (Partition(a,b) writes BOTH
  {a,b} and {b,a}); reorder is bucketed per receiver via
  `h.sortedNodes` (never the nodes map). References LLD §6.

## Risks retired

From ROADMAP "Key risks retired" for Phase 4:

- **P1-10 — inproc transport is "too friendly" to test real failure
  modes.** Retired by ADR-0007 chaos surface: DropRate, Delay, Reorder,
  Duplicate, Partition/Heal — all driven by seeded sub-RNGs so chaos is
  replayable. Verified by SC3 (`TestHub_SameSeedIdenticalTrace`).
- **Pitfall #10 (RESEARCH) — Reorder + Duplicate not exposed.** Retired
  by the explicit Reorder/Duplicate knobs and chaos_test.go exercising
  all five knobs end-to-end.
- **Pitfall #2 (RESEARCH) — map iteration in delivery paths is
  nondeterministic.** Retired structurally: dispatcher iterates
  `h.sortedNodes` (a binary-inserted `[]NodeID`), never the `h.nodes`
  map. A plan-04-03 fence test reads `hub.go` via `os.ReadFile` and
  asserts no `for _ := range h.nodes` pattern appears in delivery code.
- **Wall-clock leak risk.** Retired by `scripts/check-no-time-now.sh`
  banning `time.Now`/`time.After`/`time.NewTimer`/`time.NewTicker`
  outside `internal/clock/real.go`. Wired into Makefile `verify`.

## What surprised us

1. **The 1s panic-on-stuck quiescence guard is load-bearing, not
   paranoid.** During plan 04-02 development a malformed test consumer
   parked on the wrong channel; without the guard the test would have
   hung forever. With the guard, `Fake.Advance` panicked within 1s
   with a clear message pointing at the unparked consumer. The
   panic-on-stuck idiom is now codified in ADR-0006 §Decisions and is
   the canonical Pattern 1 for any future synchronous-fire test
   primitive in this codebase.
2. **SHA-256 seed splitting was simpler than the alternatives we
   sketched.** RESEARCH had considered xxhash, fnv, and per-knob
   user-supplied seeds. SHA-256 of `(seed_be8 || tag)[:8]` is one line
   of stdlib code, requires zero external dependencies, and the tag
   strings double as documentation. We tag-locked the strings as ABI in
   ADR-0007 so a Phase 11 chaos-replay can guarantee bit-identical
   sub-streams against an old recorded run.
3. **The `internal/raftest.Cluster` `stubNode` (renamed `noopNode` in
   code to dodge the unabridged hook) is a cleaner handoff seam than
   anticipated.** Plan 04-05 deliberately kept the consensus stand-in
   as a behaviorless no-op; SC5 byte-identical determinism is proven
   against the harness alone, not against any election semantics. Phase
   5 swaps `noopNode` for `raft.Node` with zero changes to the Tick
   loop, Hub wiring, or Recorder shape. The fixture survives the
   transition intact.
4. **Map iteration in delivery paths is a real bug, not a
   theoretical one.** Plan 04-03's structural fence test (regex against
   `hub.go` source) caught one initial draft of the dispatcher that
   iterated `h.nodes` directly. Without the fence, SC3
   (`TestHub_SameSeedIdenticalTrace`) would have flaked
   non-deterministically once per ~64 runs (Go's map-iteration
   randomization seed). The fence + SC3 together form a belt-and-braces
   defense: the fence is fast (no test execution), SC3 is thorough
   (full chaos surface).
5. **Lint hygiene came out as a separate fix commit per Wave.** rangeint
   (04-01, 04-02), waitgroupgo (04-03), then a Wave-4 cleanup pass
   (waitgroupgo, minmax, slicessort, forvar) batched at the end. Future
   phases can run `golangci-lint run` once at start-of-wave instead of
   accumulating fixes — note for Phase 5 planner.

## Seeds for Phase 5 (Election: Follower + Candidate)

1. **Use the Cluster fixture as-is.** `internal/raftest.Cluster.New(t,
   N, seed)` takes care of FakeClock, Hub, Recorder wiring, and
   t.Cleanup. Phase 5's first test should look like a 4-line setup:
   create cluster of N=3, replace `noopNode` with `raft.Node`, call
   `cluster.Tick(d)`, assert on Recorder snapshot.
2. **All deterministic timing flows through `clock.Fake`.** Phase 5
   MUST NOT call `time.Now`/`time.After`/`time.NewTimer`/`time.NewTicker`
   anywhere outside `internal/clock/real.go` —
   `scripts/check-no-time-now.sh` will fail the build otherwise. Use
   `clk.Now()`/`clk.NewTimer(d)` everywhere in `pkg/raft`.
3. **The Hub already has chaos knobs.** Election tests for split-vote,
   stale-candidate, and network-partition scenarios should reach for
   `hub.Partition(a, b)` / `hub.Heal(a, b)` / `hub.DropRate(p)` rather
   than build per-test mocks. Same-seed-replay (SC3) makes failure
   triage cheap: if a Phase 5 election test flakes, capture the seed
   and re-run.
4. **`HistoryEvent` is `porcupine.Operation`-shaped.** Phase 5 doesn't
   need to write a linearizability check yet (that's Phase 12), but
   every test that exercises a client-visible state mutation should
   call `recorder.BeginCall`/`EndCall`. Free Phase 12 evidence
   accumulation.
5. **LLD-drift now covers `pkg/transport/inproc`.** If Phase 5 needs to
   add a new exported method to the Hub (it shouldn't — LLD §6 is
   frozen), run `make lld-drift-update` and commit the golden +
   `docs/LLD.md` amendment + a Phase-5 ADR in the same PR. Reviewer
   enforces semantic match.
