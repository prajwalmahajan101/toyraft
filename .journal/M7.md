## Decisions made (non-ADR)

- **Harness driver model (ii) over (i).** `internal/raftest` drives each real
  `raft.Node` SYNCHRONOUSLY via `TickForTest`/`DrainForTest` (the exact
  production runTicker+applyOne sequence run inline) and drains inbound through
  the public `Step` — restoring Phase-5 synchronous determinism while
  exercising the real driver logic + public surface. Model (i)
  (own-driver+FakeClock) was implemented and proven functionally correct but is
  NOT settle-able within the chaos wall-clock budget (a single
  `FakeClock.Advance` cannot flush the heartbeat→delivery→Step→commit→apply
  pipeline across the async goroutines); the plan explicitly authorised the
  (ii) fallback. Result: TestFigure8 ~0.3 s; 1000-seed log-divergence sweep
  ~43 s (faster than the old 83 s harness).
- **Single-node (N=1) liveness (07-03, Rule-2).** `becomeCandidateLocked` wins
  on the self-vote quorum and `tickLeaderLocked` runs
  `maybeAdvanceCommitLocked` each heartbeat so a self-only-quorum leader
  commits; N≥3 behaviour provably unchanged (current-term Figure-8 guard
  intact).
- **`ErrStopped` single-sentinel broadening.** One sentinel now covers both the
  pre-start and post-Stop guards; `errors.Is(err, ErrStopped)` stays the lone
  classifier. §4 of LLD.md back-patched to match.

## Risks retired

Roadmap "Key risks retired": pitfalls #7 (Apply blocks loop), #8 (goroutine
leak), L-1..L-5, C-4 (panic crash), C-5 (lock across send via copy-under-lock).

- **#7 / C-4 (Apply blocks loop / panic crash)** — `applyOne` recovers the
  consumer panic, logs the stack via slog, flips fatal-status, and the loop
  keeps draining; `appliedIdx` advances on the panic path too. Evidence:
  `TestApplyPanic`.
- **#8 (goroutine leak)** — `goleak.VerifyNone` after `Stop` reports zero
  residual; Stop deadline 5 s default. Evidence: `TestStopGoroutineLeak`,
  `TestIdempotentLifecycle` (each lifecycle method ×concurrent).
- **C-5 (lock across send)** — `runTicker` drops the lock before `Send`;
  `Status` copies under lock then releases. Evidence: `-race` whole-tree green;
  SC5 ordering (`SaveHardState` before every `Send`) held.
- **L-1..L-5 (lifecycle hazards)** — idempotent `Start`/`Stop`, bounded
  `applyCh`, separate `enqueuedIdx`/`appliedIdx` frontiers, silent default
  logger. Evidence: `TestIdempotentLifecycle`, `TestSilentDefault`,
  `TestStatusLatency`, `TestApplyChannelBounded`.

## What surprised us

- **The chaos wall-clock budget killed the "pretty" harness.** The clean design
  was an own-driver harness on `FakeClock` — but a single `Advance` cannot push
  the full async pipeline to quiescence in one logical instant, so the
  wall-clock settle ballooned TestFigure8 to 30–40 s and made the 1000-seed
  sweep impractical. The synchronous in-line driver (model ii) is "uglier"
  conceptually but is BOTH deterministic AND faster (43 s vs the old 83 s).
- **Off-golden exported methods are a real escape hatch.** `TickForTest` /
  `DrainForTest` are exported, yet because their receiver `*nodeImpl` is
  unexported, `go doc -all` omits them — letting `internal/raftest` drive the
  real node without growing the public LLD surface by a single symbol.
- **The whole-tree pre-commit vet forces commit fusion.** Deleting `TestNode`
  while `raftest` still referenced it produced an intermediate state the
  whole-tree vet hook rejects, so the delete + rewire + test-rewrites had to
  land in one commit (same constraint hit in 07-01).
